Installation

This section details some of the methods of installing and accessing the Shabad OS database. There are 2 officially supported release formats (SQLite file and npm package), and a REST API provided by GurbaniNow.

tip

Remember, never modify any existing content of the database. If there is a mistake, please file an issue on GitHub. The only exception to this rule is that tables or data that you do not require may be dropped to save storage space in constrained environments.

SQLite#

The SQLite3 database can be downloaded from GitHub Releases, and follows the schema described further on.

These releases follow the semantic versioning format, so it is possible to download an updated database without breaking your application.

Running SELECT * FROM Lines ORDER BY order_id LIMIT 100 will return the first 100 lines that the database contains, in the correct order.

Examples of useful and common queries can be found in the usage section.

npm#

The database is packaged as an npm module. To install it, run npm install @shabados/database. Importing the package will give you access to the location of the database

This package uses the same semantic versioning format as the SQLite database, so running npm update will allow you to retrieve compatible database corrections without breaking your application. Additionally, there is a Javascript API that can be imported from the package, allowing you to leverage the database using wrapper functions, without the need to write SQL.

It also exposes the path of the SQLite database, which can be used directly, if desired.

For more usage instructions, see usage.

GurbaniNow API#

GurbaniNow has teamed up with Shabad OS to provide the most accurate, accessible, and transparent Gurbani database. GurbaniNow's latest v3 API is powered by the Shabad OS database, for those who wish to access Gurbani through an online REST interface. The endpoint is https://api.gurbaninow.com. See their GitHub for more information.
